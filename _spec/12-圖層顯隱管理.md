# 12 - 圖層顯隱管理

## 1. 概述

所有數據圖層（imageSets, pointSets, polylineSets, polygonSets, geojsonSets, contourSets）
統一整合至 `items` 陣列，透過 `Checkboxs` 組件提供使用者切換各圖層顯隱。

---

## 2. items 資料結構

每個 item 包含：

```js
{
    name: '圖層名稱',           // 來自各集合的 title
    msg: '圖層說明',             // 來自各集合的 msg
    order: null,                // 排序用數字
    visible: true,              // 是否可見
    updatePath: 'pointSets.0.visible' // 回寫路徑
}
```

---

## 3. 整合流程 (changeItems)

```
1. 遍歷 imageSets → 加入 items
2. 遍歷 pointSets → 加入 items
3. 遍歷 polylineSets → 加入 items
4. 遍歷 polygonSets → 加入 items
5. 遍歷 geojsonSets → 加入 items
6. 遍歷 contourSets → 加入 items
7. 排序：
   - 有 order 的項目 → 依 order 升序
   - 無 order 的項目 → 放在後面，保持原始順序
8. cloneDeep 脫勾
9. 存入 vo.items
```

---

## 4. 顯隱同步 (updateItems)

使用者切換 Checkboxs 時觸發：

```js
each(vo.items, (v) => {
    set(vo, v.updatePath, v.visible)      // 同步內部資料
    set(vo.opt, v.updatePath, v.visible)  // 同步外部 opt
})
```

### updatePath 格式

- `imageSets.{k}.visible`
- `pointSets.{k}.visible`
- `polylineSets.{k}.visible`
- `polygonSets.{k}.visible`
- `geojsonSets.{k}.visible`
- `contourSets.{k}.visible`

---

## 5. 程式化控制 (modifyItemsVisible)

外部可透過 ref 呼叫批次修改：

```js
this.$refs.map.modifyItemsVisible((item, k) => {
    return item.name.includes('溫度')
})
```

同樣會同步內部與外部 opt。

---

## 6. 觸發時機

`changeItemsDebounce` 在以下情況被呼叫：

- `changeOpt` (全域設定變更)
- `changeImageSets` (影像數據變更)
- `changePointSets` (點數據變更)
- `changePolylineSets` (折線數據變更)
- `changePolygonSets` (多邊形數據變更)
- `changeGeojsonSets` (GeoJSON 數據變更)
- `changeContourSets` (等值線數據變更)

均使用 debounce 機制避免頻繁觸發。
